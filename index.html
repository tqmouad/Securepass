Built with security as the architecture, not an afterthought.

🔑

Separate Master Password

Your encryption key derives from a master password that never touches the server. Login credentials and vault keys are separate by design.

🛡️

AES-GCM Encryption

Every password is encrypted client-side using AES-256-GCM via the browser's Web Crypto API before transmission. The server stores only ciphertext.

🎲

Cryptographic Generation

Passwords are generated using crypto.getRandomValues() — never Math.random() — so every character is genuinely unpredictable.

🗄️

Row Level Security

Supabase enforces at the database engine level that users can only access their own rows — not application-layer filtering, actual Postgres policy.

📊

Strength Analysis

Real entropy calculation — not a simple length check — gives you an honest assessment of how long your password would resist a brute-force attack.

📤

Portable Vault

Export your vault as encrypted JSON anytime. You own your data and can migrate or back it up without depending on this service to exist forever.
